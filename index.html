Cascata, Especificidade e Combinadores:

  • A metodologia de desenvolvimento de software cascata (ou waterfall) surgiu em meados dos anos 70 como uma forma de otimizar a gestão desse tipo de projeto. Baseia-se em uma hierarquia rígida de etapas sequenciais, com um controle fixo das funções.
  • A especificação é a maneira de como os navegadores definem quais valores de propriedades são os mais relevantes para o elemento a ser utilizado. A especificação é baseada apenas nas regras impostas na composição de diferentes tipos de seletores.
  • Em programação, um combinador é uma função de ordem superior que combina outras funções ou elementos sem a necessidade de variáveis, permitindo criar comportamentos mais complexos a partir de peças simples e reutilizáveis.

Box Model e Display:

  • O Box Model (Modelo de Caixa) é um conceito central no CSS que descreve como cada elemento HTML é renderizado numa página web como uma caixa retangular, composta por quatro partes: Conteúdo, Padding (preenchimento interno), Border (borda) e Margin (margem externa).
  • O funcionamento de um display envolve o controle de uma matriz de pixels ou elementos luminescentes que exibem informações através de uma interface de comunicação entre o microcontrolador (como um Arduino) e o chip controlador do display.
